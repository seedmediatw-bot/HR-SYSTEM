# HR 自動化系統建置教學（初學者版）

> 這份文件說明如何從零開始，用 AI 幫你建立一套公司 HR 系統，不需要寫程式背景。

---

## 一、這個系統是什麼？

你最終會得到一個**公司內部網頁**，員工可以用瀏覽器打開，進行：

- 📅 請假申請
- 💰 報帳申請
- 🧑‍💼 員工入職登記
- 🕐 上下班簽到
- 📊 審核進度追蹤
- 🔐 公司帳號密碼管理

所有資料都會自動存進 **Notion**，主管可以在 Notion 上審核。

---

## 二、系統運作邏輯（先看懂再動手）

```
你的電腦 / 手機
    │  打開網址
    ↓
GitHub Pages
（免費放網頁的地方，就像雲端硬碟，但放的是網頁）
    │  點按鈕送出資料
    ↓
Cloudflare Worker
（中間人，負責把資料傳給 Notion，同時藏住你的 API 金鑰）
    │
    ↓
Notion
（真正儲存資料的地方，你可以在這裡看到所有申請）
```

**為什麼需要 Cloudflare Worker？**
因為 Notion 需要一把「鑰匙」才讓你存資料，這把鑰匙不能直接放在網頁裡（任何人都看得到），所以用 Cloudflare 當中間人，幫你把鑰匙藏起來。

## 四、如何對 AI 下指令？（指令邏輯）

這個系統是透過對話讓 Claude 幫你完成的。下指令不需要懂程式，但有幾個邏輯要掌握：

### ✅ 讓 AI 先看懂你的東西

**邏輯：** 每次開始新任務前，先讓 AI 讀你現有的檔案或畫面，它才知道你的環境是什麼樣子。

> 例：「幫我看一下這個資料夾內的檔案，分析是否有問題」

---

### ✅ 說清楚你「看到什麼」和「想要什麼」

**邏輯：** 描述現象 + 說出目標，不需要解釋技術細節。

> 例：「按鈕按下去沒有反應，我希望它能送出資料到 Notion」

---

### ✅ 截圖比文字更快

**邏輯：** 遇到問題或想要複製某個效果時，直接截圖給 AI 看，比描述更精準。

> 例：（貼上截圖）「幫我做一個長這樣的介面」

---

### ✅ 說限制條件

**邏輯：** 如果有不想動的地方，提前說清楚，AI 只會改你指定的部分。

> 例：「新增這個功能，但嚴禁改寫底層邏輯」

---

### ✅ 一次做一件事

**邏輯：** 每次只請 AI 完成一個明確目標，做完確認沒問題再進行下一步，比較容易找出問題在哪。

---

### STEP 2｜取得 Notion API 金鑰

1. 前往 [notion.so/my-integrations](https://www.notion.so/my-integrations)
2. 點「New integration」，填名稱（例如：HR系統）
3. 複製「Internal Integration Secret」開頭為 `ntn_` 的那串文字，這就是你的 **Notion Token**

---

### STEP 3｜部署 Cloudflare Worker

Cloudflare Worker 是你的「中間人」，負責把 API 金鑰藏起來。

1. 前往 [workers.cloudflare.com](https://workers.cloudflare.com) 登入（免費帳號即可）
2. 點「Workers & Pages」→「Create」→「Start with Hello World」
3. 取一個名字，點「Deploy」
4. 部署完成後點「**Edit Code**」
5. 把編輯器裡所有的程式碼**全部刪掉**，貼上 `cloudflare-worker.js` 的內容
6. 點「**Save and Deploy**」

**設定環境變數（金鑰）：**
1. 進入 Worker → 點「Settings」→「Variables and Secrets」
2. 新增 `NOTION_TOKEN` = 你的 `ntn_xxxx` 金鑰
3. 點「Deploy」儲存

**確認是否成功：**
打開 `https://你的worker名稱.workers.dev/api/status`，應該看到 `{"notion": true}`

---

### STEP 4｜設定 HTML 連線到你的 Worker

打開 `index.html`，找到這一行：
```javascript
var API_BASE = 'https://...workers.dev';
```
改成你的 Worker 網址。

---

### STEP 5｜上傳到 GitHub 並發布網頁

**第一次上傳：**
```bash
git init
git add index.html .gitignore .github/
git commit -m "初始部署"
git branch -M main
git remote add origin https://github.com/你的帳號/HR-SYSTEM.git
git push -u origin main
```

**啟用 GitHub Pages：**
1. 進入 GitHub repo → Settings → Pages
2. Source 選「**GitHub Actions**」
3. 等 1-3 分鐘，你的網址就是：`https://你的帳號.github.io/HR-SYSTEM`

---

### STEP 6｜設定密碼保護

系統內建入口密碼，在 `index.html` 搜尋你的密碼數字即可修改，修改後 `git push` 即生效。

---

### STEP 7｜日後新增功能

每次修改完 `index.html`，執行：
```bash
git add index.html
git commit -m "說明改了什麼"
git push
```
GitHub Pages 約 1-2 分鐘後自動更新。

## 八、金鑰存放位置

| 金鑰 | 放在哪裡 |
|---|---|
| Notion Token | Cloudflare Worker → Settings → Variables → `NOTION_TOKEN` |
| Claude API Key | 同上 → `CLAUDE_API_KEY`（選填） |
| 本機測試用 | `.env` 檔（不會上傳到 GitHub） |

> GitHub 上看到的 `index.html` 裡面沒有任何金鑰，金鑰只存在 Cloudflare。

## 九、常見問題

**Q：按鈕按下去沒有反應？**
A：確認網頁是用 `https://` 打開的，不能用 `file://` 直接開 HTML 檔案。

**Q：Notion 連線失敗？**
A：進入 Cloudflare Worker → Settings → Variables，確認 `NOTION_TOKEN` 有填且正確。

**Q：更改密碼？**
A：在 `index.html` 搜尋現有密碼數字，換掉後 `git push`。

**Q：想新增欄位或功能？**
A：截圖給 Claude 看，說「幫我在 XX 頁面新增 OO 欄位，同步到 Notion」，Claude 會幫你修改程式碼。

**Q：審核追蹤沒有資料？**
A：點頁面上的「↻ 重新整理」，確認 Notion 裡已有資料且 Integration 有連接到該資料庫。

**Q：手機版跑版？**
A：系統已支援手機自適應，點左上角 ≡ 可開啟選單。若仍有問題，截圖給 Claude 說「手機版這個地方跑版了」。
